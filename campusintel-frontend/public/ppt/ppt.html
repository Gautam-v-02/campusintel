02 — Architecture

Three loops, one system.

"This is not a pipeline. The execution path changes based on what the agent observes at runtime."

🔵 DATA INPUT
Student submits debrief → Gemini extracts topics → college_company_intel → reactor.js listens
🟣 AGENT LOOP
Scanner detects drive → runAgentLoop() → 9 steps execute → agent_logs ★Realtime → Frontend streams live
🟠 LEARNING
TPC records outcome → learner.js → strategy_weights → Next run reads new weights
SHARED STATE — SUPABASE
colleges users campus_drives agent_logs ★ strategy_weights ★ debriefs intel + 5 more
MULTI-TENANT DESIGN
Every table carries college_id. Add a new college = one DB row. Local intel is isolated by default. Global fallback pool is opt-in. The agent always knows which campus it is reasoning for.
03 — Agentic AI Proof
The ReAct loop. Tool arbitration, not a script.
OBSERVE_PROFILE ← fetch student + drive in parallel ↓ COLD_START_CHECK ├── no data → coldStartProbe() → EXIT └── has data → continue ↓ QUERY_LOCAL_DB ← college_id + company_id ├── count ≥ 5 → use local data └── count < 5 → QUERY_GLOBAL_DB ├── count ≥ 10 → blend 70/30 └── count < 10 → SCRAPE_EXTERNAL ↓ ASSESS_READINESS score = Σ(skill_i × freq_i) / Σ(freq_i) ↓ SELECT_STRATEGY ← reads strategy_weights ├── ≥ 0.7 HIGH → BRIEF_ONLY ├── ≥ 0.4 MEDIUM → BRIEF_ASSESS └── < 0.4 LOW → BRIEF_ASSESS_SESSION ↓ GENERATE_BRIEF ← always GENERATE_ASSESSMENT ← if ASSESS in strategy SCHEDULE_SESSION ← if SESSION in strategy ALERT_TPC ← if cohort gap ≥ 3 students UPDATE_STUDENT_STATE ← closes the loop
strategy_weights — LIVE RL TABLE

WHY THIS IS AGENTIC
Tool selection is dynamic. The agent decides which tools to call based on runtime observations.
Branching decisions are logged with their decision_basis — every decision is verifiable.
Closed learning loop: outcomes update weights; next run reads updated weights. Epsilon-greedy RL.
COLD START — HANDLED
If student has no data: agent fires coldStartProbe() — sends a targeted WhatsApp asking 3 calibration questions. Response triggers loop restart. Zero state = a demo moment, not a failure state.
